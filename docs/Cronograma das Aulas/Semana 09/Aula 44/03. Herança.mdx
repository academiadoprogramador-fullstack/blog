## Introdução

**Herança** é um dos pilares da programação orientada a objetos, e permite que uma **classe derive de outra**, **reaproveitando** e **especializando comportamentos e atributos**.

- Uma **classe base (ou superclasse)** define membros comuns.
- Uma **classe derivada (ou subclasse)** herda esses membros e pode adicionar ou modificar comportamentos.
### Exemplo com Animais:

Vamos modelar o conceito de herança com uma classe base `Animal` e algumas subclasses:

```cs
public class Animal
{
    public string Nome { get; set; }

    public void Comer()
    {
        Console.WriteLine($"{Nome} está comendo.");
    }

    public void Dormir()
    {
        Console.WriteLine($"{Nome} está dormindo.");
    }
}
```
`Animal.cs`

Agora, classes que herdam de `Animal`:

```cs
public class Cachorro : Animal
{
    public void Latir()
    {
        Console.WriteLine($"{Nome} está latindo.");
    }
}

public class Gato : Animal
{
    public void Miar()
    {
        Console.WriteLine($"{Nome} está miando.");
    }
}
```
`Cachorro.cs`, `Gato.cs`
### Uso:

```cs
Cachorro cachorro = new Cachorro();
cachorro.Nome = "Rex";
cachorro.Comer();     // Herdado de Animal
cachorro.Latir();     // Específico de Cachorro

Gato gato = new Gato();
gato.Nome = "Mimi";
gato.Dormir();        // Herdado de Animal
gato.Miar();          // Específico de Gato
```
`Program.cs`

### Por que utilizar herança?

- **Reutilização de código**: métodos e propriedades comuns são definidos uma única vez.
- **Organização lógica**: permite representar relações do mundo real (ex: "um gato é um animal").
- **Extensibilidade**: novas classes podem ser criadas com base nas existentes.
